# 소요 시간 기준 최적화 리스트

가장 오래 걸리는 구간부터 손보면 체감 시간을 크게 줄일 수 있습니다. **위에서부터 순서대로** 적용하는 것을 권장합니다.

---

## 1. 코스 스캔 (검출의 ~87%, 최우선)

| # | 항목 | 내용 | 적용 방법 | 비고 |
|---|------|------|-----------|------|
| 1-1 | **저해상도 코스 스캔** | 코스 스캔만 640 등으로 줄여 프레임 추출·파이프·판정 부담 감소 | `--coarse-scale 640` (또는 960, 1280) | 실측: 1분 48초 → 약 30초 수준. 품질 동일 사례 있음. |
| 1-2 | **코스 간격 완화** | 시점 수를 줄여 추출 횟수 감소 (경계 정밀화가 보정) | `--coarse-interval 60` (기본 30) | 긴 영상에서 체감 큼. 정밀도는 소폭 감소 가능. |
| 1-3 | **GPU 디코딩** | ffmpeg 디코딩을 GPU로 이전 | `--cuda` | NVIDIA 드라이버 필요. 원본이 네트워크/HDD일 때 효과 큼. |
| 1-4 | **워커 수** | 코스 스캔 병렬화 | `--workers N` (기본 CPU 코어 수) | I/O/디코딩 병목이면 과도한 워커는 비효율. |

**주의**: `--coarse-scale 640` 사용 시 **`--coarse-cache`는 쓰지 말 것**. 캐시 쓰기만 늘어나고 경계에서는 원본이 필요해 캐시를 못 써서 전체가 느려짐.

---

## 2. 병합 단계 (--merge 실행 시 전체 시간)

| # | 항목 | 내용 | 적용 방법 | 비고 |
|---|------|------|-----------|------|
| 2-1 | **출력·임시를 로컬로** | concat·구간 쓰기를 로컬 SSD로 해서 I/O 단축 | `--merge "D:\로컬경로\merged.mp4"`, 임시 폴더를 로컬(NVMe 등)에 | 네트워크 쓰기 제거 시 효과 가장 큼. |
| 2-2 | **구간별 병렬 추출** | 구간 cut을 동시에 처리 | `--merge-workers N` (기본 2, 로컬 NVMe면 4~8 시도) | 원본이 네트워크/HDD면 2~3 권장. |
| 2-3 | **최종 파일만 네트워크로** | 병합 결과는 로컬에 두고, 필요 시 한 번만 복사 | 워크플로: 로컬에 `--merge` 실행 후 robocopy 등으로 공유 폴더 복사 | 병합 실행 시간에서 “복사” 제외. |

---

## 3. 경계 정밀화 (검출의 ~12~20%)

| # | 항목 | 내용 | 적용 방법 | 비고 |
|---|------|------|-----------|------|
| 3-1 | **정밀도 완화** | 이진 탐색 반복을 일찍 멈춰 추출 횟수 감소 | `--boundary-tolerance 1.0` (기본 0.9375, 더 완화 시 1.5 등) | 경계 시각이 최대 약 ±0.5초 단위로 흐려질 수 있음. |
| 3-2 | **이진 탐색 반복 상한** | 매우 긴 구간에서만 반복 수 제한 (이미 구현되어 있을 수 있음) | 코드에서 최대 반복 횟수 상한 확인·조정 | 극단적 긴 구간에서만 영향. |
| 3-3 | **캐시로 경계 ffmpeg 감소** | 원본 해상도 코스 시 캐시를 켜고, 경계에서 “가까운 시각”이면 캐시 사용 | `--coarse-cache` (단, **`--coarse-scale` 없이**), 캐시 폴더는 **로컬** | 저해상도와 동시에 쓰면 경계에서는 캐시 미사용이라 비추. |
| 3-4 | **경계도 저해상도(실험)** | 경계 이진 탐색에서도 640 등으로 추출·판정 | 옵션 미구현 시 `--boundary-scale 640` 등 설계·구현 | 정확도 트레이드오프. 실험용. |

---

## 4. 기타 (이미 적용했거나 환경에 따라)

| # | 항목 | 내용 | 적용 방법 | 비고 |
|---|------|------|-----------|------|
| 4-1 | **CuPy(GPU 판정)** | `--cuda` 시 판정 연산도 GPU | `pip install cupy-cuda12x` 후 `--cuda` | 병목이 디코딩·I/O면 체감은 제한적. |
| 4-2 | **원본이 로컬** | 가능하면 검출·병합 모두 로컬 디스크에서 수행 | 원본을 로컬 SSD로 복사 후 실행 | 네트워크 고정이면 2번(출력 로컬·병렬)이 우선. |
| 4-3 | **메타(ffprobe)** | 한 번 호출이라 비중 거의 없음 | - | 별도 최적화 불필요. |

---

## 5. 우선 적용 요약 (체크리스트)

- [ ] **검출 빨리**: `--coarse-scale 640` (+ 필요 시 `--coarse-interval 60`, `--cuda`, `--workers N`)
- [ ] **경계 단축**: `--boundary-tolerance 1.0` (또는 1.5), 원본 해상도 코스일 때만 `--coarse-cache`(캐시는 로컬)
- [ ] **병합 빨리**: `--merge` 출력을 **로컬 경로**로, `--merge-workers 2`(또는 로컬 NVMe면 4~8), 결과만 필요 시 네트워크로 복사

상세 근거·실측·트레이드오프는 `PIPELINE_AND_OPTIMIZATION.md` 참고.
