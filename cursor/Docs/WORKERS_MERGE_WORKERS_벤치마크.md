# Workers / Merge-Workers 벤치마크 및 튜닝 가이드

`--workers`와 `--merge-workers` 옵션에 따른 성능 비교 결과와 권장값을 정리한 문서입니다.  
동일 원본·동일 옵션으로 재실행해 비교한 로그를 바탕으로 갱신합니다.

---

## 1. 옵션 역할 요약


| 옵션                      | 역할                                             | 병목                                                      |
| ----------------------- | ---------------------------------------------- | ------------------------------------------------------- |
| `**--workers N**`       | 코스 스캔·경계 정밀화 시 병렬 프로세스 수. 기본값: CPU 코어 수(또는 8). | 같은 영상을 N개 프로세스가 동시에 읽음 → **디스크/네트워크 I/O**               |
| `**--merge-workers N`** | 병합 단계에서 구간별 ffmpeg 추출에 쓰는 프로세스 수. 기본값: 2.      | 같은 원본을 N개 프로세스가 동시에 읽고, 각자 임시 파일에 쓰기 → **디스크/네트워크 I/O** |


**무조건 많다고 빠르지 않음.** 디스크(특히 HDD·네트워크 드라이브)가 병목이면 워커를 늘릴수록 오히려 느려질 수 있음.

---

## 2. `--workers` 비교 결과

- **원본:** `\\192.168.8.113\ffmpeg\e000e77_20260227_012216.ts` (약 1시간 18분, 158시점, 삼분할 구간 3개)
- **공통 옵션:** `--cuda --coarse-scale 640 --merge-workers 2 --segments-out --merge`
- **변경:** `--workers` 2 / 4 / 8 / 16

### 2.1 요약 표


| workers | 코스 스캔     | 경계 정밀화   | 검출 전체     | 파일 병합 | 검출+병합 (대략) |
| ------- | --------- | -------- | --------- | ----- | ---------- |
| 16      | 27.3초     | 6.4초     | 33.7초     | 13.1초 | ≈ 47초      |
| **8**   | **10.7초** | **4.4초** | **15.1초** | 5.1초  | **≈ 20초**  |
| 4       | 14.7초     | 4.2초     | 18.9초     | 5.0초  | ≈ 24초      |
| 2       | 21.4초     | 7.2초     | 28.6초     | 2.0초  | ≈ 31초      |


### 2.2 해석

- **workers 8**이 이 환경에서 **가장 빠름** (검출 15초대, 전체 약 20초).
- **workers 16**은 8보다 **느림** (코스 스캔 27초 vs 10.7초) → 네트워크 드라이브에서 16개 프로세스가 동시에 같은 파일을 읽어 I/O 병목으로 추정.
- workers 4는 8보다 수 초 느림, workers 2는 가장 느림 (병렬 이득 감소).

### 2.3 권장 (현재 환경: 네트워크 원본)

- **권장:** `--workers 8`
- **비권장:** `--workers 16` (이 원본에서는 불리)
- 대안: 부하가 크면 `--workers 4`

## 4. 환경별 권장값 요약


| 저장 위치      | `--workers` | `--merge-workers` |
| ---------- | ----------- | ----------------- |
| 네트워크 / HDD | 4~8 (8 권장)  | 2                 |
| SATA SSD   | 4~8         | 2~4               |
| 로컬 NVMe    | 8~16        | 4~8               |


*실제 최적값은 원본 길이·해상도·디스크 상태에 따라 달라질 수 있으므로, 위 벤치마크 방식을 참고해 동일 원본으로 2~3번만 바꿔 가며 비교하는 것을 권장.*

---

## 5. 벤치마크 방법 (로그 비교용)

1. **한 번에 하나만 변경:** `--workers`만 바꾸거나 `--merge-workers`만 바꿔서, 어느 쪽이 체감에 영향을 주는지 구분.
2. **비교할 로그 위치:**
  - 실행 명령(1번 줄): `--workers N`, `--merge-workers M`
  - 소요 시간 요약: **코스 스캔**, **경계 정밀화**, **전체 실행**(검출)
  - 로그 맨 아래: **병합 소요 XX초**(파일 병합)
3. **전체 소요:** 검출 전체 + 병합 소요 = 대략적인 벽시계 시간.
