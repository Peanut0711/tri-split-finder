# 삼분할(Tripartite) 구간 검출 — 가능성 및 구현 방안

## 1. 문제 정의

- **입력**: FHD(1920×1080) 영상 파일 (예: `.ts`, 2시간 등)
- **검출 대상**: 화면 정중앙 1/3을 양옆으로 복제해 붙인 "삼분할" 구간
- **출력**: 해당 구간의 시작·종료 타임스탬프 (예: `00:32:30.000 ~ 00:34:30.000`)를 로그로 출력
- **제약**: 삼분할 구간은 **연속성**을 가짐 — 한 번 시작되면 **최소 약 20초** 이상 유지됨

---

## 2. 감지 원리 (가능성)

삼분할 화면의 특징은 다음과 같다.

| 구역 | 내용 |
|------|------|
| 왼쪽 1/3 | 중앙 1/3과 **동일(또는 극히 유사)** |
| 중앙 1/3 | 원본 영상 |
| 오른쪽 1/3 | 중앙 1/3과 **동일(또는 극히 유사)** |

따라서 **프레임 하나만** 있어도 다음으로 판단할 수 있다.

1. 프레임을 **세로로 3등분** (각 640×1080 픽셀)
2. **왼쪽 vs 중앙**, **오른쪽 vs 중앙** 픽셀(또는 특징) 유사도 계산
3. 두 쌍 모두 **특정 임계값 이상 유사** → 해당 프레임은 삼분할 프레임
4. 시간축으로 연속된 삼분할 프레임 구간 → **삼분할 구간**으로 보고 시작/끝 시간 기록

즉, **영상에서 일정 간격으로 프레임을 뽑아서 위 유사도만 계산하면 이론적으로 충분히 검출 가능**하다.

---

## 3. 한계·주의사항

- **인코딩/압축**: 왼·중·오가 완전 동일한 픽셀이 아니라, 인코딩으로 인해 미세한 차이가 날 수 있음 → **유사도(SSIM, MSE 등)** 사용 필요, 단순 픽셀 동일 비교는 부적절.
- **자막/로고**: 중앙에만 자막이 있고 좌우는 복제라면, 복제 시 자막이 같이 복제되었을 가능성이 높음 → 기존 로직에 큰 영향 없을 가능성이 큼. 반대로 좌우에만 다른 요소가 있다면 오탐 가능성 있음.
- **해상도/비율**: 1920×1080 기준으로 3등분(640px)이 맞음. 다른 해상도는 비율로 1/3 잘라서 비교하면 됨.

---

## 4. 추천 기술 스택

| 역할 | 도구 | 이유 |
|------|------|------|
| 프레임 추출 | **ffmpeg** | `-ss`(시작 시점) + `-i`로 빠른 구간 점프, `.ts` 지원, 특정 시간에 1프레임만 추출 가능 |
| 영상 메타(길이 등) | **ffprobe** | 길이·프레임레이트 등으로 샘플링 시점 계산 |
| 이미지 처리·유사도 | **Python + OpenCV / NumPy** | 프레임을 3등분해 픽셀/블록 비교, SSIM 등 구현 용이 |
| (선택) GPU | CUDA/OpenCV CUDA | 5070 Ti 있으면 대량 프레임 비교 시 가속 가능, 초기에는 CPU만으로도 가능 |

**정리**: **ffmpeg로 특정 시점의 프레임만 추출** → **Python에서 3등분 후 유사도 계산**이 가장 현실적이고, 구현·디버깅이 쉽다.

---

## 5. 효율적인 탐색 전략 (연속성 활용)

조건: **삼분할 구간은 한 번 나오면 최소 약 20초 연속**이다.

### 5.1 2단계 탐색

1. **1단계 — 코스 스캔(희소 샘플링)**  
   - 전체 영상 길이를 **N초 간격**(예: 5초)으로 나눔.  
   - 각 시점에서 **1프레임만** 추출해 삼분할 여부 판정.  
   - 2시간 = 7200초 → 5초 간격이면 **1440프레임**만 보면 됨 (매 프레임 디코딩하는 것보다 훨씬 적음).

2. **2단계 — 구간 경계 정밀화**  
   - 코스 스캔에서 "이전 샘플은 삼분할 O, 다음 샘플은 삼분할 X"인 **경계**를 찾음.  
   - 그 경계 구간(예: 5초 구간) 안에서만 **더 촘촘히**(1초 간격 또는 이진 탐색) 샘플링해 **시작/끝 시간을 1초 단위(또는 원하는 정밀도)로 좁힘**.

이렇게 하면:

- **대부분의 시간은 5초마다 1프레임만** 검사하고,
- **경계 근처에서만** 추가로 몇 프레임을 더 보므로,
- 전체 디코딩·연산량을 크게 줄일 수 있다.

### 5.2 (참고) 말씀하신 “25초 후 없으면 20초부터 1초씩 뒤로” 전략

- 코스 스캔에서 “마지막 삼분할 O” 시점을 **T**라고 하면,  
  T+5초에서 “삼분할 X”가 나왔다고 가정.
- 그러면 **끝 구간**은 [T, T+5] 사이에 있음.
- 여기서 **T+1, T+2, T+3, T+4** 초 각각 1프레임씩 보면서 “마지막 삼분할 프레임”을 찾으면, 끝 시점을 1초 단위로 정할 수 있음.  
  (T+20초부터 1초씩 “뒤로” 가는 것과, T부터 1초씩 “앞으로” 가는 것은 같은 구간을 다른 방향으로 스캔하는 것이므로, 구현 시 **끝 경계 구간 [T, T+5] 안에서 1초 단위 스캔**으로 통일하면 됨.)

---

## 6. 유사도 지표 제안

- **MSE (Mean Squared Error)**  
  - 왼쪽 1/3 vs 중앙 1/3, 오른쪽 1/3 vs 중앙 1/3 각각 계산.  
  - MSE가 **임계값 이하**이면 “같은 내용”으로 간주.  
  - 구현 간단, 속도 빠름.

- **SSIM (Structural Similarity)**  
  - 구조적 유사도라서 인코딩 노이즈에 더 강함.  
  - OpenCV(`cv2.SSIM`) 또는 `scikit-image`로 계산 가능.  
  - 정확도는 좋지만 MSE보다 연산이 조금 더 듦.

**권장**: 먼저 **MSE**로 프로토타입을 만들고, 오탐/미탐이 있으면 **SSIM**으로 바꾸거나 둘을 조합(예: MSE와 SSIM 모두 임계 통과 시만 삼분할로 판정).

## 7. 구현 흐름 (의사코드)

```
1. ffprobe로 영상 길이 D(초) 획득
2. 코스 스캔: t = 0, 5, 10, ... D-5 초에 대해
   - ffmpeg로 시점 t에서 1프레임 추출 (예: PNG 또는 raw → NumPy)
   - 프레임을 세로 3등분 → left, center, right
   - (left, center), (right, center) 유사도 계산
   - 둘 다 임계값 이상이면 "tripartite(t) = True", 아니면 False
3. tripartite 시퀀스에서 "True → False" 또는 "False → True" 전환 구간 찾기
   → 각 전환 구간 [t_start_cand, t_end_cand] (예: 5초 구간)
4. 경계 정밀화: 각 [t_start_cand, t_end_cand] 안에서
   - 시작 경계: 1초(또는 0.5초) 단위로 앞/뒤로 스캔해 "처음 삼분할이 나온 시점" 찾기
   - 끝 경계: 같은 방식으로 "마지막 삼분할이 나온 시점" 찾기
5. 인접 구간 병합 (선택): 끝 시점과 다음 구간 시작 시점이 매우 가깝면 하나로 합침
6. 최종 구간 리스트를 로그로 출력 (예: "00:32:30.000 ~ 00:34:30.000")
```

## 8. 요약

| 항목 | 내용 |
|------|------|
| **가능 여부** | **가능** — 프레임 3등분 후 좌/중, 우/중 유사도만 보면 됨 |
| **추천 방식** | ffmpeg로 특정 시점 1프레임 추출 + Python(OpenCV/NumPy)으로 3등분·유사도 계산 |
| **탐색 효율** | 5초 간격 코스 스캔 → 경계 구간에서만 1초 단위 정밀 스캔 (연속성 활용) |
| **유사도** | MSE로 시작, 필요 시 SSIM 추가 |
| **출력** | 시작·종료 타임스탬프를 로그(print 또는 파일)로 출력 |

이 방안을 기준으로 `tripartite_section_check.py`에 **입력: 영상 경로(예: input.ts), 출력: 삼분할 구간 타임스탬프 로그** 형태로 구현하면 된다.  
구현 시에는 **임계값(유사도)**과 **코스 스캔 간격(5초)**을 인자로 두어, 실제 영상으로 튜닝하기 쉽게 두는 것을 권장한다.
