# Tri-Split Detector 전체 동작 흐름

현재 `tri_split_detector.py`의 동작을 단계별로 정리한 문서입니다.

---

## 1. 실행 진입점

```bash
python tri_split_detector.py <비디오파일경로.ts>
```

- 인자로 `.ts` 영상 경로 1개를 받습니다.
- 인자가 없으면 사용법을 출력하고 종료합니다.

---

## 2. 전체 흐름 개요

```
[시작]
    │
    ▼
┌─────────────────────────────────────┐
│ 1. 초기화 (SplitVideoDetector)      │  ← ffprobe로 메타정보만 조회, 영상 인덱싱 없음
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│ 2. find_first_tri_split()            │
│    - 30초 간격으로 프레임 스캔       │
│    - 각 시점마다 FFmpeg로 1프레임만 추출
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│ 3. 삼분할 후보 프레임 발견 시        │
│    - 분할선 X 좌표 계산              │
│    - 디버그 이미지/그래프 저장       │
│    - 이진 탐색으로 정확한 시작 시각 역추적
└─────────────────────────────────────┘
    │
    ▼
[반환: (시작시간, 프레임인덱스, left_x_end, right_x_start) 또는 None]
```

---

## 3. 단계별 상세

### 3.1 초기화 (`__init__`)

| 항목 | 내용 |
|------|------|
| **입력** | `video_path`, 옵션으로 `y_start=540`, `y_end=720` (Y 안전 영역) |
| **ffprobe 호출** | `format=duration`, `stream=r_frame_rate` 로 영상 길이(초)와 fps만 조회. **전체 프레임 인덱싱 없음.** |
| **저장 값** | `self.video_path`, `self.fps`, `self.total_frames`(≈ duration×fps), `self.y_start`, `self.y_end` |

초기 로딩이 거의 0초에 가깝게 끝나도록, decord 대신 ffprobe만 사용합니다.

---

### 3.2 프레임 추출 (`_get_frame_at_index`)

| 항목 | 내용 |
|------|------|
| **역할** | 특정 프레임 인덱스에 해당하는 **1장의 이미지**만 가져옵니다. |
| **방식** | `frame_idx` → `timestamp_sec = frame_idx / fps` 로 변환 후, **FFmpeg** subprocess 호출. |
| **FFmpeg 옵션** | `-ss <초>` 를 `-i` 앞에 두어 **input seeking** (빠른 구간 이동), `-vframes 1` 로 1프레임만 출력. |
| **결과** | 임시 PNG 저장 → `cv2.imread` → BGR→RGB 변환 후 **numpy 배열** 반환. |

영상 전체를 메모리에 올리지 않고, 필요한 시각만 요청하는 구조입니다.

---

### 3.3 첫 삼분할 탐색 (`find_first_tri_split`)

#### 3.3.1 스캔 루프

- **점프 간격**: 30초 (`jump_frames = 30 * fps`).
- **동작**: `frame_idx = 0` 부터 `total_frames` 미만까지, 30초마다 한 번씩:
  1. `_get_frame_at_index(frame_idx)` 로 해당 시각의 1프레임 획득.
  2. Y 안전 영역만 잘라 `safe_frame` 생성.
  3. **해상도 1/4 축소**: `safe_small = cv2.resize(..., 1/DETECT_SCALE)` 로 연산량 감소.
  4. **삼분할 여부 판별** (아래 3.3.2).

삼분할이 아니면 `frame_idx += jump_frames` 하고 다음 30초 시점으로 진행합니다.

#### 3.3.2 삼분할 판별 (Template Matching)

- **가정**: 좌측·중앙·우측이 비슷한 폭으로 나뉜 화면에서, **좌측과 우측이 같은 영상**이면 삼분할로 봅니다.
- **방법**:
  1. **템플릿**: 축소된 프레임의 **왼쪽 영역** `[0 : 600/4]` (원본 기준 0~600 픽셀).
  2. **탐색 영역**: **오른쪽** `[1260/4 : 1900/4]` (원본 1260~1900).
  3. `cv2.matchTemplate(search_area, template, cv2.TM_CCOEFF_NORMED)` 로 상관계수 계산.
  4. **일치율 ≥ 0.95** 이면 해당 프레임을 “삼분할 후보”로 간주합니다.

#### 3.3.3 분할선 X 좌표 계산 (삼분할로 판정된 경우)

1. **우측 분할 시작 X (`right_x_start`)**  
   - 템플릿 매칭에서 최대 일치 위치 `max_loc[0]` (축소 좌표)를 원본으로 복원:  
     `right_x_start = 1260 + max_loc[0] * DETECT_SCALE`

2. **좌측 분할 끝 X (`left_x_end`)**  
   - 축소 영역에서 **좌측 영역** `[0 : cw_small]` 과 **우측 영역** `[rs : rs+cw_small]` 의 픽셀 차이(`col_diff`) 계산.  
   - 좌측 분할선은 **640±20 (620~660)** 안에 있다고 가정하고, 해당 구간에서 `col_diff > 15.0` 인 첫 위치를 찾아 원본 좌표로 변환:  
     `left_x_end = (search_start + mismatch_indices[0]) * DETECT_SCALE` (등).

#### 3.3.4 디버그 출력 (선택)

- **디버그 이미지**: 원본 해상도 프레임에  
  - Y 안전 영역(초록 박스), 템플릿 매칭 영역(파란 박스), 분할선(빨간 선), 시간·좌표 텍스트를 그려 `picture/debug_frame_<시간>.jpg` 로 저장.
- **오차 그래프**: `col_diff` 를 matplotlib으로 그려 `picture/debug_graph_<시간>.png` 로 저장. (matplotlib 없으면 메시지만 출력)

#### 3.3.5 이진 탐색 (정확한 시작 시각)

- **목적**: 30초 단위로 찾은 “삼분할 구간” 안에서, **삼분할이 실제로 시작되는 프레임**을 약 1초 단위까지 좁힙니다.
- **구간**: `[frame_idx - jump_frames, frame_idx]` (최대 30초 구간).
- **조건**: 구간 폭이 `fps` 이하가 될 때까지 반복.
  - 중간 인덱스 `mid_idx` 의 프레임을 `_get_frame_at_index(mid_idx)` 로 가져옴.
  - 같은 방식으로 1/4 축소 후 template matching.
  - 일치율 ≥ 0.95 이면 “아직 삼분할” → `search_right_idx = mid_idx`, 그렇지 않으면 `search_left_idx = mid_idx`.
- **결과**: 최종 `exact_start_idx` 에 대해 `exact_time_str = format_time(exact_start_idx / fps)` 를 계산해 “대략적인 삼분할 시작 시간”으로 출력.

---

## 4. 출력·반환

- **콘솔**:  
  - 첫 삼분할 발견 시각, 좌표 `(left_x_end, right_x_start)`, 디버그 저장 경로, 역추적된 시작 시각, 총 탐색 소요 시간.
- **반환값**:  
  - 발견 시: `(exact_time_str, exact_start_idx, left_x_end, right_x_start)`  
  - 미발견 시: `None`

---

## 5. 의존성·환경

| 항목 | 설명 |
|------|------|
| **외부 명령** | `ffprobe`, `ffmpeg` 가 PATH에 있어야 합니다. |
| **Python 패키지** | `numpy`, `opencv-python` (cv2). |
| **선택** | `matplotlib` 이 있으면 오차 그래프를 저장합니다. |
| **해상도** | 연산 가속을 위해 `DETECT_SCALE = 4` (1/4 해상도)로 matchTemplate/col_diff 수행. 원본 해상도는 디버그 이미지와 최종 좌표 복원에만 사용. |

---

## 6. 요약

- **초기화**: ffprobe로 길이·fps만 조회 → 인덱싱 없이 즉시 시작.
- **스캔**: 30초 간격으로 FFmpeg가 해당 시각 1프레임만 추출 → 1/4 축소 후 template matching으로 삼분할 여부 판별.
- **발견 시**: 분할선 X 계산 → 디버그 저장 → 이진 탐색으로 시작 시각 역추적 → 시간·좌표 반환.

이 문서는 현재 코드 기준이며, `tri_split_guide.md`, `속도 개선안.md` 와 함께 참고하면 됩니다.
